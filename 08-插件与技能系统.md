# 插件与技能系统

## 1. 插件系统 (plugins/)

### 1.1 插件加载机制

Claude Code 的插件系统支持内置插件和第三方插件。

### 1.2 内置插件 (builtinPlugins.ts)

内置插件与 Claude Code CLI 一起打包，可以通过 `/plugin` UI 启用/禁用。

**与 Bundled Skills 的区别**：
| 特性 | 内置插件 | Bundled Skills |
|------|----------|----------------|
| 出现位置 | `/plugin` UI | `/skills` 列表 |
| 用户控制 | 可启用/禁用 | 始终可用 |
| 组件类型 | 多种组件 | 仅 skills |

### 1.3 插件注册和执行流程

注册流程：启动时调用 `initBuiltinPlugins()` 注册所有内置插件

### 1.4 钩子事件类型

- PreToolUse / PostToolUse - 工具执行前/后
- PermissionDenied - 权限被拒绝
- SessionStart/SessionEnd - 会话开始/结束
- SubagentStart/SubagentStop - 子智能体启动/停止
- TaskCreated/TaskCompleted - 任务创建/完成

## 2. 技能系统 (skills/)

### 2.1 技能定义和格式

技能定义包含：name, description, aliases, whenToUse, argumentHint, allowedTools, model 等。

### 2.2 技能执行机制 (SkillTool)

两种执行模式：
1. **Inline 模式** - 技能内容直接注入对话
2. **Fork 模式** - 在独立子智能体中运行技能

### 2.3 内置技能

| 技能名 | 描述 |
|--------|------|
| remember | 记忆技能 |
| loremIpsum | 生成占位文本 |
| verify | 验证技能 |
| loop | 循环执行技能 |
| claudeApi | Claude API 相关 |
| debug | 调试技能 |

### 2.4 用户自定义技能

用户可以在 `~/.claude/skills/` 目录下创建自定义技能。

## 3. MCP 集成

技能系统支持 MCP 技能构建器。

## 4. 权限与安全

### 4.1 技能白名单

SAFE_SKILL_PROPERTIES 定义了安全属性列表，只包含这些属性的技能会自动获得权限。
