# 服务层详解

## 概述

服务层是 Claude Code 的核心业务逻辑层，提供 API 通信、协议管理、数据处理等功能。

## 1. services/api/ - Anthropic API 客户端

### client.ts - API 客户端工厂

创建配置好的 Anthropic API 客户端实例。

**支持的认证方式**:
- 直接 API 访问
- AWS Bedrock
- Azure Foundry
- Google Vertex AI

### claude.ts - API 调用封装

核心功能包括流式查询模型、获取最大输出 token、解析 prompt 太长错误等。

## 2. services/mcp/ - Model Context Protocol

### 类型系统

配置作用域：local, user, project, dynamic, enterprise, claudeai, managed

传输类型：stdio, sse, sse-ide, http, ws, sdk

### useManageMCPConnections.ts - 连接管理

核心职责：
- 初始化 MCP 客户端连接
- 设置连接生命周期处理器
- 管理自动重连

## 3. services/oauth/ - OAuth 2.0 认证

### OAuthService 类

实现 OAuth 2.0 授权码流程（支持 PKCE）。

**两种授权流程**:
1. 自动流程 - 打开浏览器完成授权
2. 手动流程 - 显示手动授权 URL

## 4. services/lsp/ - Language Server Protocol

### manager.ts - LSP 管理器

管理 LSP 服务器实例，提供代码导航、诊断等功能。

## 5. services/analytics/ - 分析服务

### 核心功能

- 事件日志记录
- GrowthBook 特性开关
- 事件队列管理

## 6. services/plugins/ - 插件系统

### pluginOperations.ts - 插件操作

核心操作：安装、卸载、启用/禁用、更新插件。

## 7. services/compact/ - 对话压缩

### compact.ts - 核心压缩逻辑

支持完整压缩和部分压缩两种类型。

## 8. services/policyLimits/ - 组织策略限制

获取并应用组织级别的策略限制。

## 9. services/remoteManagedSettings/ - 远程托管设置

为企业客户获取和管理远程托管设置。

## 10. services/extractMemories/ - 自动记忆提取

从当前会话中提取持久记忆并写入自动记忆目录。

## 11. services/tokenEstimation.ts - Token 估算

估算文本和消息的 token 数量。

## 服务层架构图

```
CLI 入口 → 命令层 → 服务层 → 工具层
```

服务层包含：API 客户端、MCP、OAuth、LSP、Analytics、Plugins、Compact、PolicyLimits 等。
