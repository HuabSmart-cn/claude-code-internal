# 配置与迁移系统

## 目录结构

```
src/
├── schemas/           # Zod 配置模式定义
├── migrations/        # 配置迁移脚本
└── utils/settings/   # 设置管理系统
```

## 1. 配置模式 (schemas/)

使用 Zod 定义所有配置项的验证模式。

## 2. 设置管理 (utils/settings/)

### 2.1 设置加载优先级 (从低到高)

1. 默认值
2. 托管设置文件 (managed-settings.json)
3. 托管设置 Drop-in 目录
4. 用户设置文件 (~/.claude/settings.json)

### 2.2 变更检测

检测设置项的变化，生成变更事件。

### 2.3 验证

使用 Zod 进行运行时参数校验。

## 3. 迁移系统 (migrations/)

### 3.1 迁移文件列表

| 文件 | 用途 |
|------|------|
| migrateAutoUpdatesToSettings | 自动更新迁移 |
| migrateBypassPermissionsAcceptedToSettings | 绕过权限迁移 |
| migrateEnableAllProjectMcpServersToSettings | MCP 服务器迁移 |
| migrateFennecToOpus | Fennec → Opus 模型迁移 |

### 3.2 迁移执行时机

1. 应用启动时
2. 设置加载前
3. 版本检测后

## 4. 托管设置系统

### 4.1 Drop-in 机制

```
~/.claude/settings.json           # 用户设置
/etc/claude/managed-settings.json  # 系统托管基础
/etc/claude/managed-settings.d/
  ├── 10-base.json
  ├── 20-security.json
  └── 30-otel.json
```

### 4.2 合并规则

1. 基础文件先加载
2. Drop-in 文件按文件名排序后加载
3. 后加载的文件优先级更高

## 5. 配置验证规则

### 5.1 路径验证

禁止危险路径 (.., /etc/, /sys/)。

### 5.2 命令验证

白名单检查。

### 5.3 模型名称验证

验证模型名称是否在允许列表中。
