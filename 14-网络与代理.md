# 网络与代理

## 目录结构

```
src/
├── upstreamproxy/     # 上游代理配置
├── remote/           # 远程会话
└── server/           # 服务器模式
```

## 1. 上游代理 (upstreamproxy/)

### 1.1 代理核心

管理 Claude Code 与上游服务之间的 HTTP/SOCKS 代理连接。

**核心功能**:
- 代理协议支持 (HTTP, HTTPS, SOCKS4, SOCKS5)
- 代理认证 (Basic, NTLM, Negotiate)
- 连接池管理
- 自动重试和故障转移

### 1.2 代理中继

在代理链中转发请求，支持多层代理。

## 2. 远程会话 (remote/)

### 2.1 会话管理器

管理远程 Claude Code 实例的会话：
- 会话生命周期管理
- 连接状态监控
- 消息路由
- 断开重连

### 2.2 WebSocket 会话

通过 WebSocket 管理远程会话的实时通信。

### 2.3 权限桥接

在远程会话中桥接权限检查。

## 3. 服务器模式 (server/)

### 3.1 直连会话

创建直接的点对点连接会话。

### 3.2 直连管理器

管理所有直连会话：
- 会话注册/注销
- 连接状态跟踪
- 资源清理

## 4. 连接流程

### 4.1 远程会话建立

```
Client --- WebSocket Connect ---> Server
Client <-- Handshake Challenge ----
Client --- Handshake Response ---->
Client <-- Session Created -------
Client <==== Session Active ======>
Client <-- Session Terminated -----
```

### 4.2 代理请求流程

```
Claude Code → UpstreamProxy → Anthropic API
```

## 5. 安全考虑

- TLS/SSL 支持
- Token-based 认证
- 定期心跳检测
- 审计日志记录
