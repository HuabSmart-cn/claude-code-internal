# Claude Code 桥接系统详解

## 概述

桥接系统是 Claude Code CLI 与 IDE 扩展（VS Code, JetBrains）之间的双向通信层。

## 核心架构

```
IDE Extension (VS Code / JetBrains)
         │
         │ WebSocket / HTTP
         ▼
Bridge Layer (bridgeMain.ts, bridgeMessaging.ts, etc.)
         │
         ▼
Claude Code CLI (子进程)
```

## 1. bridgeMain.ts - 桥接主入口

核心类型：BridgeState, ReplBridgeHandle

### 桥接生命周期

1. 注册桥接环境
2. 创建会话
3. 启动工作轮询循环
4. 处理入口消息
5. 清理

## 2. bridgeMessaging.ts - 消息处理

### 消息入口处理

处理 IDE 和 CLI 之间的消息路由。

### 可桥接消息过滤

过滤出可以在桥接中传输的消息类型。

## 3. replBridge.ts - REPL 会话桥接

### 传输类型

1. HybridTransport (v1) - WebSocket 读取，HTTP POST 写入
2. SSETransport (v2) - Server-Sent Events 读取

## 4. jwtUtils.ts - JWT 认证

### JWT 解析

解析 JWT payload 用于认证。

### Token 刷新调度器

在 token 过期前 5 分钟自动刷新。

## 5. sessionRunner.ts - 会话执行管理

### 子进程管理

管理 Claude Code CLI 子进程的生命周期。

## 6. bridgePermissionCallbacks.ts - 权限回调

处理桥接中的权限请求和响应。

## 7. 权限请求流程

```
CLI → BridgePermissionCallbacks → IDE Extension
                    ↑                          ↓
                    └────── 用户响应 ←─────────┘
```

## 相关源码文件

| 文件 | 功能 |
|------|------|
| bridgeMain.ts | 桥接主入口和生命周期管理 |
| bridgeMessaging.ts | 消息处理、路由、流量控制 |
| replBridge.ts | REPL 会话桥接 |
| sessionRunner.ts | 子进程会话管理 |
| jwtUtils.ts | JWT 解析和刷新调度 |
| bridgePermissionCallbacks.ts | 权限回调接口 |
