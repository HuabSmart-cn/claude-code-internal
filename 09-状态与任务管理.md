# 状态与任务管理

## 1. 状态管理 (state/)

### 1.1 状态存储机制

Store 模式实现：支持 getState、setState、subscribe。

特性：不可变性、订阅机制、变更回调。

### 1.2 AppState 结构

AppState 是全局状态的核心结构，包含设置与配置、任务管理、MCP 状态、插件状态、Agent 定义、团队上下文、推理状态等。

### 1.3 状态持久化

- 设置存储在 `~/.claude/settings.json`
- 任务状态存储在 `~/.claude/tasks/`
- 团队配置存储在 `~/.claude/teams/`

## 2. 任务管理 (tasks/)

### 2.1 任务类型

- LocalShellTaskState - 本地 Shell 任务
- LocalAgentTaskState - 本地 Agent 任务
- RemoteAgentTaskState - 远程 Agent 任务
- InProcessTeammateTaskState - 进程中队友任务

### 2.2 任务状态机

```
pending → in_progress → completed
    ↓           ↓           ↓
  blocked    blocked     blocked
    ↓           ↓
  cancelled  cancelled
```

### 2.3 任务工具

- TaskCreateTool - 创建任务
- TaskUpdateTool - 更新任务状态
- TaskListTool - 列出任务
- TaskStopTool - 停止任务

### 2.4 任务间依赖

通过 blocks 和 blockedBy 字段管理任务依赖关系。

## 3. 后台任务管理

### 3.1 背景任务识别

通过 isBackgroundTask 函数识别后台任务。

### 3.2 任务与 Hooks 的集成

支持 TaskCreated 和 TaskCompleted 钩子。
