# Claude Code 权限系统详解

## 概述

Claude Code 的权限系统管理工具调用的授权流程。当 Claude 尝试执行危险操作时，系统会请求用户确认。

## 核心架构

```
权限请求入口 → 权限模式检查 → 权限处理器 → 权限决策
```

## 1. 权限模式 (PermissionMode)

```typescript
type PermissionMode =
  | 'default'        // 标准交互式权限确认
  | 'plan'           // 计划模式，只读操作
  | 'bypassPermissions'  // 跳过所有权限检查
  | 'auto'           // 自动批准基于分类器
  | 'localAssistant' // 本地助手模式
```

## 2. PermissionContext - 权限上下文

管理工具信息、日志记录、权限持久化、决策构建、用户处理等。

## 3. 权限决策类型

```typescript
type PermissionDecision =
  | { behavior: 'allow'; updatedInput?: Record<string, unknown>; ... }
  | { behavior: 'deny'; message: string; ... }
  | { behavior: 'ask' }
  | { behavior: 'skip' }
```

## 4. 权限处理器

### Coordinator Handler
用于协调者工作进程的权限处理。

### Interactive Handler
用于主代理的交互式权限处理。

### Swarm Worker Handler
用于 Swarm Worker 的权限处理。

## 5. 权限规则管理

### PermissionUpdate

```typescript
type PermissionUpdate = {
  destination: 'alwaysAllowRules' | 'alwaysDenyRules' | 'command'
  rule: { tool: string; input?: Record<string, unknown>; matchers?: InputMatcher[] }
  permanente: boolean
}
```

## 6. Bash 分类器

当启用 BASH_CLASSIFIER 特性时，对 Bash 命令执行异步推理来确定是否自动批准。

## 7. 权限检查入口

```typescript
export async function hasPermissionsToUseTool(
  tool: Tool,
  input: Record<string, unknown>,
  toolUseContext: ToolUseContext,
  assistantMessage: AssistantMessage,
  toolUseID: string,
): Promise<PermissionDecision>
```

## 8. 权限处理流程

1. 模式检查 - bypassPermissions 直接允许
2. 规则检查 - alwaysAllowRules/alwaysDenyRules
3. 分类器检查 - 仅 Bash 工具支持
4. Hook 检查 - 执行 PermissionRequest hooks
5. 降级到交互式确认对话框

## 关键常量

- GRACE_PERIOD_MS = 200ms - 用户交互取消分类器的宽限期
- CHECKMARK_DISPLAY_MS = 3000ms/1000ms - 分类器自动批准后显示 ✓ 的时间
